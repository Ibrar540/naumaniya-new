## Features

### Module Detection
The parser automatically detects which module the query targets:
- **Students** → طلباء, student, admission
- **Teachers** → اساتذہ, teacher, staff
- **Classes** → کلاس, class, capacity
- **Masjid Income** → مسجد آمدنی, donation
- **Masjid Expenditure** → مسجد خرچ, expense
- **Madrasa Budget** → مدرسہ بجٹ, school budget

### Filter Extraction
Supports multiple filter types:
- **Status filters**: active, graduate, struck off (English & Urdu)
- **Date filters**: year, month+year, date ranges
- **Amount filters**: exact, greater than, less than, ranges
- **Class filters**: A, B, C (English & Urdu)
- **Text filters**: name, description searches
- **ID filters**: specific record IDs

### Intent Detection
Automatically determines query intent:
- `exact_list` - Show specific records
- `filtered_list` - Show filtered results
- `total_computation` - Calculate totals/sums

### Smart Recommendations
Generates 3 contextually relevant suggestions:
- Based on detected module
- Adapts to user's language (English/Urdu)
- Updates dynamically as user types

## Technical Details

### Parser Output Format
```dart
{
  'module': 'students',
  'intent': 'filtered_list',
  'confidence_score': 85,
  'requires_pagination': false,
  'filters': {
    'id': null,
    'status': 'active',
    'class': ['A'],
    'date_filter': {...},
    'amount_condition': {...}
  },
  'total_computation': null,
  'recommendations': [
    'Suggestion 1',
    'Suggestion 2',
    'Suggestion 3'
  ]
}
```

### Conversion Flow
1. User enters query
2. `AIQueryParser.parse()` analyzes query
3. `_convertParserResult()` transforms to internal format
4. Service processes query with filters
5. Results returned with suggestions
6. UI displays results and recommendations
